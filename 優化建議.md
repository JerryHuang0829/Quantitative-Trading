### Caution exposure 為何不建議調整

**表面看**：caution 70→85% 在 3Y 有最高 Sharpe（1.89）和 Alpha（+54.37%）。
**實際問題**：2022-2024 有 78% 的月份為 caution 或 risk_off。提高 caution exposure 等於在 78% 的樣本上增加曝險。

- 波動率上升 7%（24.0%→25.7%），beta 上升 3.4%（0.524→0.542）
- 這不是更好的選股，是在回測期間恰好是牛市的時候拿更多部位
- 真正的空頭市場中，高 caution exposure 會放大損失
- **結論：嚴重 overfit 風險，不落地**

## P4：工程化

### 4.1 Paper Trading 框架（最高優先）

P1+P2 的 in-sample 優化已完成，**最重要的下一步是累積 out-of-sample 數據**驗證策略真實性。

**目標：** 每月記錄策略建議，6 個月後比較回測預期與實際表現。

**具體實作方案：**

1. **記錄機制**
   - 每月 12 號（再平衡日）自動或手動執行策略，記錄建議持股與權重
   - 輸出格式：`reports/paper_trading/YYYY-MM.json`
   - 欄位：`date`, `market_signal`, `positions[]`（symbol, name, weight, score）, `gross_exposure`

2. **績效追蹤**
   - 用下個月 12 號的收盤價計算持有期間報酬
   - 累積月報酬序列，計算 rolling Sharpe / Alpha / MDD
   - 輸出：`reports/paper_trading/performance.json`（逐月累積）

3. **比較基準**
   - 與回測 artifact 對照：3Y Sharpe 1.85、Alpha +48.43%
   - 警戒線：Sharpe 差 > 50%（即 < 0.93）或 Alpha 轉負 → 觸發診斷
   - 正常偏差範圍：短期（< 6 個月）Sharpe 波動 ±0.5 屬正常

4. **時間表**
   - 第 1-3 個月：累積數據，不做判斷
   - 第 4-6 個月：初步趨勢觀察
   - 第 7 個月起：可初步評估是否接近回測預期

**啟動前提：**
- Docker bot 能正常執行 `docker compose up -d portfolio-bot`
- `.env` 已設定 `FINMIND_TOKEN`
- 確認 `config/settings.yaml` 為最新 P2 設定

## P3 後補充建議（2026-03-31）

### 1. 如果 `quality` 暫不落地，就不要讓它增加資料成本

目前最務實的做法是：

- 只在 `score_weights["quality"] > 0` 時才呼叫 `fetch_financial_quality()`

這樣才能讓「quality 關閉」真正等於：

- 不影響排序
- 也不影響 API / cache / 執行面

### 2. baseline artifact 要固定單一路徑

目前正式 baseline 實際在：

- `reports/backtests/p2_if0/`

建議後續固定：

- 研究基準都從這個路徑讀
- 或把它升成根目錄 canonical artifact

避免之後複核者誤讀根目錄較差的 scratch run。

### 3. 對 `quality` 的下一步不是直接放棄，而是先修財務定義

若未來還要重做品質因子，建議先改成更像投資研究口徑的版本，例如：

- TTM net income / average equity
- 或至少明確區分單季、累計、年化口徑

在這之前，不建議再花時間調 `quality_raw` 的 0.6 / 0.4 權重。
