## 3. 策略深度分析

### 3.1 因子計算評價

| 因子 | 權重 | 計算方式 | 評價 |
|------|------|----------|------|
| price_momentum | 45% | 3M×0.20 + 6M×0.35 + 12M-1M×0.45 | ✅ 經典 cross-sectional momentum，skip 最近月避 short-term reversal |
| trend_quality | 20% | close vs SMA20/60 + structure + regime，最高 1.0 | ⚠️ 離散二元計分，多頭市場所有股票全滿分，percentile 無區分力 |
| revenue_momentum | 25% | YoY×0.70 + 加速度×0.30，35 天延遲 cutoff | ✅ Look-ahead 保護正確，YoY + 加速度組合合理 |
| institutional_flow | 10% | 連續 3 天外資/投信淨買超 → ±80/90 | ⚠️ 僅 5 個離散值（-90,-80,0,+80,+90），>50% 時間 = 0，小型股永遠 = 0 |

### 3.2 Eligibility Filter 過於嚴格（核心問題）

`tw_stock.py:658-670` 的篩選條件（必須**全部同時**滿足）：

| 條件 | 門檻 | 空頭影響 |
|------|------|----------|
| close > min_price | 20 元 | 低 |
| avg_turnover_20 > min_avg_turnover | 5000 萬 | 中 |
| close > SMA60 | — | **高：空頭幾乎全部不過** |
| SMA20 > SMA60 | — | **高：空頭幾乎全部不過** |
| momentum_6m > 0 | — | **高：空頭幾乎全部不過** |
| 歷史資料 >= 274 天 | — | 中 |

**結果：** 2022 台股大跌（加權指數 18000→12000），幾乎沒有股票能同時滿足。
3Y 回測前 18 個月平均只有 **2.1 檔**持股（目標 8 檔），其中 8 個月只有 **1 檔**（6238 欣興）。

**這不是 bug，是策略設計問題。** 動量策略在空頭「沒有好標的」是正常的，但不應把所有資金壓在 1 檔小型股上。

### 3.3 隱含的集中度風險

| 場景 | 持股行為 | 問題 |
|------|---------|------|
| risk_off + 1 eligible | 12% 單一持倉 + 88% 現金 | 無分散、整年績效看 1 檔 |
| risk_on + 3 eligible | 3×12% = 36% + 64% 現金 | 大量現金拖累報酬 |
| risk_on + 8 eligible | 8×12% = 96% + 4% 現金 | ✅ 正常運作 |

`max_position_weight: 12%` 保護了單股不過大，但代價是 eligible 不足時大量現金閒置。

### 3.4 trend_quality 區分力不足

`tw_stock.py:1063-1083` 用 0/1 式二元加總：

```
close > sma_slow → +0.35
close > sma_fast → +0.15
sma_fast > sma_slow → +0.25
structure == 1 → +0.15
trending_up → +0.10
```

多頭市場大多數股票全滿分 → percentile rank 全部 = 0.5 → **20% 權重形同浪費**。

### 3.5 institutional_flow 有效性存疑

- 僅 5 個離散值，>50% 時間 = 0
- 10% 權重但對選股幾乎無影響
- 小型股永遠 = 0（免費 API 無法人資料，~44% 股票受影響）
- **實質上是一個隱性 size tilt**

## 6. 已確認做對的地方

- Ranking 只在 eligible universe 內做 percentile
- 交易日判斷依真實市場資料，fail-closed
- 月營收有 look-ahead 保護（35 天延遲 cutoff）
- 安全閥：分析成功率 < 30% 時跳過再平衡
- `_DataSlicer`：OHLCV / institutional / revenue / market_value 全部截斷至 as_of
- `backtest_start` + `reference_now` 傳入 slicer，coverage 與跨日漂移已修
- 失敗 API 呼叫以哨兵快取，避免跨再平衡重試
- Pre-filter evenly spaced，僅在 size proxy fallback 路徑啟用
- Symbol filter `(?:\d{4}|00\d{4})`，live/backtest 一致，ETF 可控，71xxxx 權證阻擋
- KPI 為日頻序列，交易成本首次建倉 + 每次再平衡均正確扣除
- 交易成本模型合理：round-trip 0.47% + slippage 5bps
- Turnover threshold 6 分：新股 score 需高出被替換股才換，減少無效換手
- 產業集中限制：同產業最多 2 檔
- 日頻報酬 `(ret_df * w).sum(axis=1)` 正確模擬 cash drag（未投資部分報酬 = 0）
- Preflight check 已加入，回測前先驗 API 可用性
- institutional / revenue source 層已修正為保留 API window 全量資料
- slicer 層 `_truncate_by_date_col` 已改為依 unique dates 截斷

## 7. Open Limitations

- `FinMindSource` 查詢仍以 `datetime.now()` 為終點（非 fetch-as-of）
- Holdings 為 weight-based，非 share-based（paper trading 前 hard blocker）
- Benchmark 為單一價格報酬代理（非含息總報酬）
- 空頭市場 eligible universe 極度縮小（策略設計層級問題）
- 3Y 回測 trading_days=574（期望 ~756），~24% 交易日缺失待查
