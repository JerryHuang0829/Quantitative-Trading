# 量化交易專案 — 學習筆記

## 一、這個專案在做什麼？

每個月自動從台股中挑出最強的 8 檔股票，組成投資組合，並定期換股。

用數據和公式取代人的主觀判斷 — 這就是「量化交易」。

```
📊 資料收集          用 FinMind API 抓台股數據（股價、法人、營收）
      ↓
🏗️ 建立選股池        從 2900+ 檔台股中，篩選出約 80 檔候選股
      ↓
📈 因子打分          對每檔候選股計算 3 個「因子」分數
      ↓
🏆 排名選股          按綜合分數排名，選出前 8 名
      ↓
⚖️ 決定權重          根據分數高低分配每檔持倉比例
      ↓
🔄 每月換股          每月 12 號收盤後執行換股
      ↓
📱 通知              透過 Telegram 發送換股通知
```

---

## 二、資料來源

所有資料來自 FinMind（台灣金融資料 API），主要 5 種：

| 資料類型 | 說明 | 用途 |
|---------|------|------|
| 股價 (OHLCV) | 每天的開高低收量 | 計算動能、技術指標 |
| 三大法人買賣超 | 外資、投信、自營商每天買賣 | 判斷「聰明錢」動向 |
| 月營收 | 公司每月營業額 | 判斷公司成長力道 |
| 市值 | 公司總市值 | 篩選大型股 |
| 股票基本資料 | 產業別、市場別 | 分類與排除 |

## 三、實際運行排程

### 運行模式：常駐程式 + 輪詢

`main.py` 啟動後會無限循環，每 15 分鐘檢查一次是否該換股。

**不是每月手動執行一次**，而是讓程式一直跑著，它會自己判斷時機。

### 換股觸發的 7 個條件（全部要通過）

```
條件 1 ✅ portfolio.enabled = true          → 策略有開啟
條件 2 ✅ rebalance_frequency = monthly     → 月頻換股
條件 3 ✅ 今天不是週末                       → 週六日不跑
條件 4 ✅ 今天不是假日                       → 國定假日不跑
條件 5 ✅ 今天日期 ≥ 12 號                  → 每月 12 號之後才換
條件 6 ✅ 現在時間 ≥ 14:00 且盤已收         → 收盤後才跑
條件 7 ✅ 這個月還沒換過股                   → 一個月只換一次
```

### 為什麼選 12 號？

台灣上市公司每月 10 號前公布上月營收，等到 12 號確保已拿到最新營收數據。

### 時間軸範例

```
3/1 ~ 3/11   每 15 分鐘檢查 → 全部跳過（還沒到 12 號）
3/12 上午      檢查 → 跳過（還沒收盤）
3/12 14:00後   檢查 → ✅ 全通過 → 執行換股 → 發 Telegram 通知
3/13 ~ 3/31   檢查 → 跳過（這個月已經換過了）
4/12 14:00後   ✅ 再次換股
```

### 部署方式

```bash
# Docker（推薦）
docker compose up -d portfolio-bot

# 或直接跑 Python
python main.py
```

## 四、回測績效的正確理解

### 常見誤解

```
❌ 「3 年報酬 37%」= 買了放 3 年不動
✅ 「3 年報酬 37%」= 策略連續運行 3 年，每月都在換股，累積年化報酬 37%
```

每一檔股票平均只持有 1～3 個月，不是 3 年。

### 目前績效（2026-04-02 更新）

| 期間 | 年化報酬 | Sharpe | Alpha | 性質 |
|------|---------|--------|-------|------|
| 3年 (2022-2024) | 53.57% | 1.85 | +48.43% | In-Sample（調參用） |
| 6個月 (2024下半年) | 31.78% | 1.08 | +13.52% | In-Sample |
| **2025 全年** | **42.33%** | **1.81** | **+8.16%** | **Out-of-Sample（驗證用）** |

> **In-Sample vs Out-of-Sample：**
> - In-Sample = 用來調整策略參數的時期，績效可能被美化
> - Out-of-Sample = 策略從未見過的時期，是真實績效的指標
> - OOS Sharpe 1.81 vs IS 1.85，衰減僅 2%，表示策略沒有嚴重 overfit
> - Alpha +8.16% 是扣掉 0050 報酬後的超額報酬（benchmark 已修復 stock split）

## 五、換股的意思

系統每月產出「換股建議」，**不會自動下單**，需要自己去券商 APP 操作。

### 三種情況

| 狀態 | 意思 | 你要做的 |
|------|------|---------|
| 繼續持有 (HOLD) | 上月買的排名還在前面 | 不動 |
| 新進場 (ENTER) | 新進前 8 名 | 買進 |
| 出場 (EXIT) | 掉出前 8 名 | 賣掉 |

### 防頻繁交易機制

- 新股要比舊股高 6 分以上才會替換（`turnover_score_threshold: 6.0`）
- 持有緩衝 3 名（`hold_buffer: 3`）：排第 9-11 名的舊持股也會保留

## 七、重要觀念釐清：技術指標 vs 因子選股

### 這個專案是「因子選股」，不是「技術指標交易」

```
技術指標策略（傳統）：RSI < 30 → 買！MACD 黃金交叉 → 買！
因子選股（本專案）  ：80 檔一起算分 → 排名 → 選前 8 名
```

### indicators.py 算了很多指標，但大部分沒被用到

| 指標 | 有被因子選股用到？ | 說明 |
|------|:---:|------|
| SMA（均線） | ✅ | 用在 trend_quality + regime 判斷 |
| ADX（趨勢強度） | ✅ | 用在 trend_quality + regime 判斷 |
| 市場結構（HH/HL） | ✅ | 用在 trend_quality |
| RSI | ❌ | 算了但沒被用到（舊版遺留） |
| MACD | ❌ | 算了但沒被用到 |
| 布林通道 | ❌ | 算了但沒被用到 |
| ATR | ❌ | 算了但沒被用到 |
| 乖離率 | ❌ | 算了但沒被用到 |
| RSI 背離 | ❌ | 算了但沒被用到 |

### 為什麼算了卻沒用？

專案演進過程中，從舊版（個股技術分析訊號）轉為新版（因子選股投組），`indicators.py` 保留了所有計算，但 `tw_stock.py` 只取用了 SMA、ADX、市場結構。

### 兩種方法的關鍵差異

| | 技術指標策略 | 因子選股（本專案） |
|---|---|---|
| 決策方式 | RSI < 30 → 買 | 80 檔排名，選前 8 |
| 比較對象 | 跟自己的歷史比 | 跟其他股票比（橫截面） |
| 買賣依據 | 單一指標觸發 | 多因子綜合排名 |
| 學術分類 | 技術分析 | 量化因子投資 |
| 使用者 | 散戶常用 | 機構法人常用 |

**因子選股是更「正統」的量化方法**，也是量化交易工程師日常在做的事。

## 八、程式碼逐檔解析

### 7.1 base.py — 資料介面（規格書）

**位置**：`src/data/base.py`

定義所有資料來源必須提供的功能（抽象類別）：

```
必須實作：
  fetch_ohlcv()         → 取得股價（開高低收量）
  is_market_open()      → 現在有沒有開盤

可選：
  fetch_institutional() → 法人買賣超
  fetch_month_revenue() → 月營收
  fetch_stock_info()    → 股票基本資料
  fetch_market_value()  → 市值
```

設計目的：未來換資料源（例如 Yahoo Finance）時，只要新來源也實作這些方法，其他程式碼不用改。這叫「介面抽象」。

---

### 7.2 finmind.py — 資料抓取 + 快取

**位置**：`src/data/finmind.py`

#### 快取系統（_DiskCache）

```
問題：FinMind 免費版每小時只能呼叫 600 次 API
解決：把抓過的資料存在硬碟（pickle 檔案）

  第一次抓台積電 → 呼叫 API → 存到 data/cache/ohlcv/2330.pkl
  第二次需要    → 直接讀 pkl → 不呼叫 API
  之後每天     → 只補抓新的 1-3 天資料
```

快取結構：
```
data/cache/
  ├── ohlcv/2330.pkl              台積電歷史股價
  ├── institutional/2330.pkl      台積電法人資料
  └── revenue/2330.pkl            台積電月營收
```

#### 速率限制（_rate_limit）

每次 API 呼叫間隔至少 0.5 秒，避免超過配額。

#### fetch_ohlcv 流程

```
fetch_ohlcv("2330", "D", limit=320)

  1. 先看快取有沒有
     ├ 有 → 資料夠新嗎（3天內）？
     │       ├ 夠新 → 直接用 ✅
     │       └ 不夠 → 只補抓缺少的幾天 ✅
     └ 沒有 → 第一次，一次抓 11 年（4000天），存入快取

  2. 從快取中切出需要的範圍（最近 320 天）
  3. 回傳 DataFrame
```

效能：
- 第一次跑：約 10-20 分鐘（抓 80 檔 × 11 年）
- 之後每次：約 30 秒（只補新資料）

### 7.3 indicators.py — 技術指標計算

**位置**：`src/strategy/indicators.py`

輸入原始股價（OHLCV），輸出加上 20+ 個新欄位的 DataFrame。

#### 基礎指標

| 指標 | 公式概念 | 白話解釋 |
|------|---------|---------|
| **SMA（均線）** | 過去 N 天收盤價平均 | 把股價「抹平」看趨勢。短均 > 長均 = 多頭排列 |
| **RSI** | 漲跌天數比值（0-100） | 體溫計。> 70 過熱，< 30 過冷 |
| **MACD** | 短期趨勢 vs 長期趨勢的差距 | 加速度。柱狀體放大 = 動能加速 |
| **布林通道** | 均線 ± 2 倍標準差 | 上軌=貴、下軌=便宜。%B 看位置 |
| **ATR** | 每天平均波動幅度 | 量尺。用來衡量距離的單位 |
| **ADX** | 趨勢強度（0-100） | 不管方向，> 25 有趨勢，< 20 在盤整 |
| **乖離率** | 股價偏離均線多少 % | 橡皮筋效應：拉太遠會彈回 |

#### 進階指標

| 指標 | 白話解釋 |
|------|---------|
| **RSI 背離** | 股價創新低但 RSI 沒有 → 下跌力道衰竭 → 領先反轉信號 |
| **市場結構** | 高低點排列。Higher High + Higher Low = 上升結構 = 趨勢健康 |

### 7.4 regime.py — 大盤多空判斷

**位置**：`src/strategy/regime.py`

用 0050（台灣50 ETF）判斷大盤狀態：

```
              ADX < 20？
             ╱        ╲
           是           否
           ↓             ↓
        「盤整」     ADX 20~25？
        ranging      ╱        ╲
                   是           否（≥ 25）
                   ↓             ↓
              有市場結構      看 SMA 排列
              確認嗎？        ╱        ╲
             ╱     ╲     短均>長均   短均<長均
           有       沒有     ↓          ↓
           ↓        ↓    「多頭」    「空頭」
        看方向    「盤整」  trending   trending
                           _up        _down
```

#### 大盤狀態對應持倉比例

| 狀態 | 持倉比例 | 意義 |
|------|---------|------|
| 多頭 (risk_on) | 96% 持股，4% 現金 | 趨勢向上，大膽持有 |
| 盤整 (caution) | 70% 持股，30% 現金 | 方向不明，減碼觀望 |
| 空頭 (risk_off) | 35% 持股，65% 現金 | 趨勢向下，大幅減碼 |

#### 不同市場狀態使用不同指標權重

- **多頭**：重視「回調進場」(25%) + 「結構確認」(20%) → 像衝浪，等浪來順著走
- **盤整**：重視「RSI 背離」(25%) + 「布林通道」(20%) → 像釣魚，在固定水域等反轉

### 7.5 tw_stock.py — 核心策略邏輯（大腦）

**位置**：`src/portfolio/tw_stock.py`

這是整個系統的核心，串接前面所有模組。

#### 全流程

```
run_tw_stock_portfolio_rebalance() ← 每月觸發
  │
  ├ Step 1: 判斷大盤（用 0050）
  ├ Step 2: 對 80 檔候選股各算 3 個因子
  ├ Step 3: 百分位排名 → 綜合評分
  ├ Step 4: 選股（保留舊持股 + 產業分散 + 換股門檻）
  ├ Step 5: 分配權重（score_weighted）
  └ Step 6: 輸出快照 + 通知
```

#### 三大因子（P2 落地後）

> P2 研究發現 `institutional_flow`（法人動向）的 rank IC 全期為負（-0.053），移除後 Alpha 提升且風險不增。目前使用三因子。

##### 1. price_momentum（價格動能）— 權重 55%

```
= 3個月報酬×20% + 6個月報酬×35% + 12-1個月報酬×45%

「12-1 月」= 跳過最近 1 個月，因為短期有反轉效應

範例：
  台積電：3M +8%, 6M +25%, 12-1M +40%
  → 0.08×0.20 + 0.25×0.35 + 0.40×0.45 = 0.2835

  鴻海：3M -2%, 6M +5%, 12-1M +15%
  → -0.02×0.20 + 0.05×0.35 + 0.15×0.45 = 0.081
```

##### 2. trend_quality（趨勢品質）— 權重 20%

```
均線多頭排列 + ADX 趨勢強度 + 市場結構（HH/HL）
→ 不只看漲多少，還看「漲得穩不穩」
```

##### 3. revenue_momentum（營收動能）— 權重 25%

```
月營收年增率（YoY）
→ 營收成長 = 基本面好 = 股價有支撐
→ 避免只靠技術面選到「假突破」
```

##### 已停用：institutional_flow（法人動向）— 權重 0%

```
原本：近 5 天的 70% × 外資買賣超 + 30% × 投信買賣超
停用原因：rank IC 全期為負（2022-2024），加入反而拉低選股品質
程式碼保留但權重設為 0，不影響排名也不呼叫 API
```

#### 綜合評分方式

```
所有因子先做「百分位排名」（0-100 分），再加權平均

為什麼？不同因子的數值範圍差異大：
  動能：-0.5 ~ +1.0
  法人：0 ~ 100
  直接加權會不公平 → 統一轉成排名才能比較
```

#### 篩選門檻

```
多頭/盤整時（嚴格）：
  ❌ 股價低於 60 日均線 → 淘汰
  ❌ 20日均 < 60日均 → 淘汰（多頭排列壞了）
  ❌ 6 個月動能 ≤ 0 → 淘汰
  ❌ 股價 < 20 元 → 淘汰
  ❌ 日均成交 < 5000 萬 → 淘汰

空頭時（放寬）：
  ❌ 股價低於 60日均的 85% → 才淘汰
  ❌ 6 個月動能 ≤ -15% → 才淘汰
```

#### 選股三步驟

```
Step 4a — 保留舊持股
  排名在 top_n + hold_buffer（前 11 名）且分數 ≥ 60 → 繼續持有
  避免「差一名就換」的頻繁交易

Step 4b — 產業分散
  同產業最多 3 檔
  例：半導體前 8 都上榜 → 只留前 3 檔，剩下給其他產業
  避免單一產業風險

Step 4c — 換股門檻
  新股要比被替換的舊股高 6 分以上才換
  差距不夠 → 不換，因為交易成本 0.47% 會吃掉優勢
```

#### 權重分配（score_weighted）

```
分數越高 → 分配越多資金 → 乘上市場曝險比例

範例（risk_on = 96%）：
  台積電 85分 → (85/230)×96% = 35.5%
  聯發科 75分 → (75/230)×96% = 31.3%
  廣達   70分 → (70/230)×96% = 29.2%
  現金                       =  4.0%

  單檔上限 12%，超過就壓到 12%
```

## 十、名詞對照表

| 概念 | 白話解釋 |
|------|---------|
| 因子 (Factor) | 給股票打分的「標準」，像考試的評分項目 |
| 動能 (Momentum) | 漲的繼續漲、跌的繼續跌的現象 |
| 百分位排名 | 在全班中排第幾名（轉成 0-100 分） |
| Regime | 大盤現在是牛市、熊市、還是盤整 |
| Rebalance | 定期檢查並調整持股（換股） |
| Sharpe Ratio | 每承受一份風險，能賺多少報酬（越高越好） |
| Alpha | 比大盤多賺（或少賺）多少 |
| Point-in-Time | 回測時不偷看未來資料，確保公平 |
| Slippage | 理想價格和實際成交價的差距 |
| Hold Buffer | 持股緩衝，避免排名差一點就頻繁換股 |
| Score Weighted | 分數越高的股票分配越多資金 |
| ATR | 每日平均波動幅度，當作量距離的尺 |
| ADX | 趨勢強度指標，不管方向，只看有沒有趨勢 |
| SMA | 簡單移動平均線，把股價抹平看趨勢 |
| RSI | 過熱/過冷指標，> 70 超買、< 30 超賣 |
| MACD | 動能加速度，柱狀體放大 = 趨勢加速 |
| 布林通道 | 股價的正常波動範圍，碰上軌=貴、碰下軌=便宜 |
| 乖離率 (BIAS) | 股價偏離均線多遠，像橡皮筋拉太遠會彈回 |
